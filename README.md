## 👨‍💻 About This Repository

This is not a tutorial dump. This is a **living engineering lab** — a documented, project-based journey through JavaScript from the ground up to full-stack backend development.

Every concept is reinforced with working code. Every module closes with a mini project. Every week, a learning log entry is written. The goal is not just to *know* JavaScript — it's to **think like a JavaScript engineer**.

> *"The best way to learn to code is to build things that matter to you — then understand why they work."*

---

## 🧠 Learning Philosophy

| Principle | What It Means Here |
|---|---|
| **Project-first** | Every concept is applied to a real mini project |
| **Document everything** | Learning logs capture what stuck and what didn't |
| **Struggle productively** | Exercises are attempted before looking at solutions |
| **Iterate in public** | Commits are frequent, meaningful, and traceable |
| **Depth over breadth** | Fewer topics mastered deeply beats many skimmed |

## 🏆 Real-World Projects

### 1. 📓 DevNotes — Developer Knowledge Base
A personal CLI + web knowledge base where developers save code snippets, notes, and learnings with tags and full-text search.

**Skills demonstrated:** Node.js, file system, REST API, CLI tooling, search algorithms

---

### 2. 📊 Habit Tracker API
A full-featured REST API with authentication, habit tracking, streaks, and statistics — built without frameworks to prove raw Node.js competency.

**Skills demonstrated:** HTTP from scratch, JWT auth, JSON persistence → SQLite, MVC architecture

---

### 3. 🤝 CollabBoard — Real-Time Collaboration Tool
A lightweight real-time shared whiteboard/notes tool using WebSockets.

**Skills demonstrated:** Event-driven architecture, WebSockets, state sync, client-server design

## 🧾 Commit Convention

All commits follow this convention for a clean, professional git history:

```
feat:     add new concept example or feature
docs:     update README, learning log, or roadmap
project:  add or update a mini/real-world project
exercise: add or solve a practice challenge
refactor: improve existing code quality or structure
fix:      correct bugs or broken examples
chore:    repo maintenance, restructuring
```

**Examples:**
```bash
feat: add closure examples with memoization
docs: update learning log — week 3 async/await
project: build async weather app with error handling
exercise: solve array flattening challenge
refactor: rewrite promise chain using async/await
```
